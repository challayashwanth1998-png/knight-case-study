🎯 Decisions

Why We Built It This Way

Key architectural choices and rationale

Content-Based Classification

Never uses filenames — only content. Handles "scan001.pdf" correctly every time.

Hybrid Extraction

Excel/CSV use Python regex at $0. Only CDL images use Gemini Vision. Saves ~70% on API costs.

Parallel Processing

ThreadPoolExecutor with 4 workers. Pipeline: ~45s vs ~3min sequential.

Configurable Rules

All 27 rules in DB with severity + enable/disable. Underwriters tune without code changes.

Cross-Document Conflicts

Validates vehicle/driver counts, company names, FEIN/DOT, duplicate CDLs and VINs across docs.

Team Routing

Auto-routes to Standard, Specialty, Driver, Ops, or Senior UW based on triggered rules.

Dual Intake

Web form + IMAP email. Agents email submissions without logging into any portal.

Database Portability

SQLAlchemy ORM. Switch SQLite → PostgreSQL with one connection string.

🔒 Security

Security & Governance

Audit trail, encryption, human-in-the-loop

Complete Audit Trail

Every action logged to audit_logs: uploads, classification, extraction, AI, rules, decisions, reviews.

Data Encryption

  • At rest: S3 AES-256
  • In transit: IMAP TLS, Gemini HTTPS
  • Credentials: Env vars, never in git

Human-in-the-Loop

No auto-approval. Must Approve, Reject, or Override with notes + timestamp.

AI Transparency

Tracks input/output tokens, cost, call count per submission.

Scalability

  • DB: SQLite → PostgreSQL
  • Compute: EC2 vertical or ECS horizontal
  • AI: Gemini auto-scales + rate limiting
  • Queue: SQS/Redis for async at scale
