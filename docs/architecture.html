Design Decisions
Key architectural choices and rationale

Content-Based Classification

Never uses filenames — only document content. Handles "scan001.pdf" correctly every time.

Hybrid Extraction Strategy

Excel/CSV use Python regex at $0 cost. Only CDL images use Gemini Vision. Saves ~70% on API costs.

Parallel Processing

ThreadPoolExecutor with 4 workers. Full pipeline: ~45 seconds vs ~3 minutes sequential.

Configurable Rules Engine

All 27 rules stored in DB with severity levels and enable/disable. Underwriters can tune without code changes.

Cross-Document Conflict Detection

Validates vehicle/driver counts, company names, FEIN/DOT numbers, duplicate CDLs and VINs across all documents.

Automated Team Routing

Auto-routes to Standard Review, Specialty Risk, Driver Review, Operations, or Senior UW based on triggered rules.

Dual Intake Channels

Web upload form + IMAP email integration. Insurance agents can email submissions without logging into any portal.

Database Portability

SQLAlchemy ORM abstraction layer. Switch from SQLite to PostgreSQL with one connection string change.

Security & Governance
Audit trail, encryption, human-in-the-loop review

Complete Audit Trail

Every action logged to audit_logs table: uploads, classification, extraction, AI calls, rules evaluation, decisions, and reviews.

Data Encryption

  • At rest: S3 AES-256 server-side encryption
  • In transit: IMAP over TLS, Gemini API over HTTPS
  • Credentials: Environment variables, never committed to source control

Human-in-the-Loop Review

No auto-approval of any submission. Underwriter must explicitly Approve, Reject, or Override with notes and timestamp.

AI Cost Transparency

System tracks input tokens, output tokens, API cost, and call count for every submission processed.

Production Scalability Path

  • Database: SQLite → PostgreSQL (RDS)
  • Compute: EC2 → ECS/Fargate horizontal scaling
  • AI: Gemini API auto-scales with built-in rate limiting
  • Queue: Add SQS or Redis for async processing at scale
