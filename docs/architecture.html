Data Flow

What goes in, what comes out

Six document types enter. A structured decision exits.

📥 Documents In

Insurance Application — PDF with company info, coverage requests, operational details

Driver Roster — Excel spreadsheet with names, DOB, CDL numbers, hire dates

Equipment Schedule — Excel with VINs, year, make, model, stated values

Loss Run Reports — PDF claims history with total incurred amounts

Driver Records — CDL class, endorsements, experience years, violations, age per driver

Vehicle Fleet — VIN, year, make, type, stated value per vehicle

Claims History — Open/closed claims, total incurred, loss ratio

Mileage Data — States of operation, total annual miles, IFTA compliance

Risk Assessment — Overall risk score with per-domain confidence levels

Data Conflicts — Cross-document validation issues flagged automatically

Recommendations — Premium suggestions, conditions, exclusions

Rules Results — All 27 rules evaluated: PASS / FAIL / WARNING

Decision + Routing — Accept, Refer, or Decline with team assignment

Architecture Decisions

Why we built it this way

Key choices and their rationale — each one optimizes for accuracy, cost, or speed.

Content-Based Classification

The system never uses filenames — only content. Handles "scan001.pdf" or "IMG_4521.jpg" correctly every time.

Hybrid Extraction Strategy

Excel and CSV use Python regex at zero AI cost. Only CDL images use Gemini Vision. This saves ~70% on API costs.

Parallel Processing

ThreadPoolExecutor with 4 workers runs AI calls concurrently. Pipeline: ~45s vs ~3min sequential.

Configurable Rules Engine

All 27 rules stored in DB. Adjustable severity, enable/disable flags. Underwriters tune without code changes.

Cross-Document Conflict Detection

Validates vehicle counts, driver counts, company names, FEIN/DOT, duplicate CDLs and VINs across all documents.

Smart Team Routing

Auto-routes to Standard Review, Specialty Risk, Driver Review, Operations, or Senior Underwriting based on which rules fired.

Dual Intake Channels

Web form + IMAP email. Agents can email submissions without logging into any portal.

Database Portability

SQLAlchemy ORM. Switch SQLite → PostgreSQL with one connection string change — zero code modifications.

Security & Governance

Trust, transparency, compliance

Full audit trail, data encryption, and human-in-the-loop review.

Complete Audit Trail

Every action logged to audit_logs: uploads, classification, extraction, AI analysis, rule evaluations, decisions, and underwriter reviews — all with timestamps.

Data Encryption

  • At rest: S3 AES-256 server-side encryption
  • In transit: IMAP TLS (993), Gemini HTTPS
  • Credentials: Environment variables, never in code or git

Human-in-the-Loop

No submission auto-approved. Underwriter must explicitly Approve, Reject, or Override. All actions include reviewer name, notes, and timestamp.

AI Cost Transparency

Every AI call tracks input tokens, output tokens, cost, and call count per submission. Displayed in the UI for full visibility.

Production Scalability

  • Database: SQLite → PostgreSQL (connection string change)
  • Compute: Vertical EC2 scaling or ECS/Fargate horizontal
  • AI: Gemini auto-scales; add rate limiting for cost control
  • Queue: SQS or Redis for async processing at scale
