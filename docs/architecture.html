🎯 Decisions

Design Decisions

Key architectural choices and their rationale.

1 Content-Based Classification

The system never uses filenames for classification — only content. This handles random filenames like "scan001.pdf" correctly.

2 Hybrid Extraction

Excel/CSV parsed with Python regex at zero AI cost. Only CDL images use Gemini Vision. Reduces API costs by ~70%.

3 Parallel Processing

AI calls use ThreadPoolExecutor with 4 workers. CDL images batch-processed in parallel. ~45s vs ~3min sequential.

4 Configurable Rules

All 27 rules stored in DB with adjustable severity and enable/disable flags. Underwriters tune rules without code changes.

5 Conflict Detection

Cross-document validation: vehicle/driver counts, company names, FEIN/DOT, duplicate CDLs and VINs. Flagged as warnings.

6 Team Routing

Auto-routes to Standard Review, Specialty Risk, Driver Review, Operations, or Senior Underwriting based on triggered rules.

7 Email Intake

IMAP watcher provides second intake channel. Agents email submissions with attachments — processed automatically, no login required.

8 Postgres-Ready

SQLAlchemy ORM abstracts database. Switch SQLite → PostgreSQL with one connection string change — zero code modifications.

🔒 Security

Security & Governance

Audit trail, data protection, and compliance.

1 Complete Audit Trail

Every action logged to audit_logs with timestamps: uploads, classification, extraction, AI analysis, rules, decisions, reviews.

2 Data Encryption

  • At rest: S3 AES-256 server-side encryption
  • In transit: IMAP TLS (993), Gemini HTTPS
  • Credentials: Environment variables, never in code/git

3 Human-in-the-Loop

No auto-approval. Underwriter must explicitly Approve, Reject, or Override. All actions include reviewer, notes, timestamp.

4 AI Transparency

Every AI call tracks input tokens, output tokens, cost, call count. Displayed per submission for cost visibility.

5 Scalability Path

  • DB: SQLite → PostgreSQL (connection string change)
  • Compute: Vertical EC2 scaling or ECS/Fargate horizontal
  • AI: Gemini auto-scales; add rate limiting for cost control
  • Queue: SQS/Redis for async processing at scale
