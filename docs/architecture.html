Design Decisions
Key architectural choices and their rationale

1 Content-Based Classification

The system never uses filenames for document classification — only document content. This ensures correct classification even when agents submit files with random or incorrect names (e.g., "scan001.pdf" for an insurance application).

2 Hybrid Extraction Strategy

Structured documents (Excel, CSV) are parsed with Python regex at zero AI cost. Only unstructured content (CDL images) uses Gemini Vision. This reduces API costs by ~70% compared to full-AI extraction.

3 Parallel Processing

AI calls use ThreadPoolExecutor with 4 workers for parallel risk analysis. CDL images are also batch-processed in parallel. Total pipeline time: ~45 seconds vs ~3 minutes sequential.

4 Configurable Rules Engine

All 27 business rules are stored in the database with configurable severity levels and enable/disable flags. This allows underwriters to tune rules without code changes — critical for real-world insurance operations.

5 Cross-Document Conflict Detection

The system validates data consistency across all uploaded documents — vehicle counts, driver counts, company names, FEIN/DOT numbers, duplicate CDLs, and duplicate VINs. Conflicts are flagged as warnings for manual review.

6 Team-Based Routing

Submissions are automatically routed to the appropriate team (Standard Review, Specialty Risk, Driver Review, Operations, Senior Underwriting) based on which rules triggered. This streamlines the review workflow.

7 Email Intake Channel

The IMAP watcher provides a second intake channel alongside the web form. Agents can email submissions with attachments, and the system processes them automatically — no portal login required.

8 SQLite for MVP, Postgres-Ready

SQLAlchemy ORM abstracts the database layer. The current SQLite deployment is sufficient for the demo, but switching to PostgreSQL requires only a connection string change — no code modifications.

Security & Governance
Audit trail, data protection, and compliance considerations

1 Complete Audit Trail

Every action in the system is logged to the audit_logs table with timestamps, including: file uploads, classification decisions, extraction results, AI analysis, rule evaluations, decisions, and underwriter reviews. This provides full traceability for compliance.

2 Data Encryption

  • At rest: S3 bucket uses AES-256 server-side encryption for all documents
  • In transit: IMAP uses TLS (port 993), Gemini API uses HTTPS
  • Credentials: API keys stored as environment variables, never in code or git

3 Human-in-the-Loop Review

No submission is auto-approved without human oversight. The review workflow requires an underwriter to explicitly Approve, Reject, or Override each decision. All review actions include reviewer name, notes, and timestamp in the audit log.

4 AI Transparency

Every AI call tracks token usage and cost. The system displays input tokens, output tokens, total cost, and call count per submission. This ensures cost visibility and enables monitoring for anomalous AI behavior.

5 Scalability Path

  • Database: Swap SQLite → PostgreSQL via connection string change
  • Compute: Scale EC2 instance vertically, or use ECS/Fargate for horizontal scaling
  • AI: Gemini API scales automatically; add rate limiting for cost control
  • Queue: Replace synchronous pipeline with SQS/Redis for async processing at scale
